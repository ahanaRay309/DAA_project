📐 Formal Problem Formulation
Why this matters: A formal ILP formulation proves NP-Hardness (reduces to 0/1 Knapsack), justifies DP as exact solver, and exposes trade-offs each algorithm makes.
Single-Rider ILP (0/1 Knapsack)
Objective: maximise
Σᵢ (pᵢ − penaltyᵢ) · xᵢ // net profit

Subject to:
Σᵢ tᵢ · xᵢ ≤ T // time budget
xᵢ ∈ {0, 1} // binary

pᵢ = rider profit for order i
tᵢ = baseTime[i] × zoneTrafficMult[i]
penaltyᵢ = SLA penalty (capped at ₹150)
T = total time budget
DP Recurrence — Corrected Traceback
dp[i][t] = max net-profit
using first i orders, budget t

Base: dp[0][t] = 0 ∀t

Recurrence:
skip: dp[i-1][t]
take: dp[i-1][t-tᵢ]+netᵢ if tᵢ≤t
dp[i][t] = max(skip, take)

Traceback (fixed):
// Critical: check strictly > not ≥
// to handle ties correctly
if dp[i][t] > dp[i-1][t]:
select order i
t -= tᵢ
SLA Penalty — Capped Model (Fixed)
penalty(tᵢ, SLAᵢ) =
0 if tᵢ ≤ SLAᵢ
α·(tᵢ − SLAᵢ) if tᵢ ∈ (SLA, 1.5·SLA]
min(β·e^(tᵢ/SLAᵢ), CAP) if tᵢ > 1.5·SLA

α = ₹1.5/min (linear zone)
β = ₹8 (exponential base)
CAP = ₹150 (realistic max)
SLA: Express=20, High=30, Medium=45, Low=60
FIX: Exponential without cap grew to ₹10k+
for long delays — unrealistic. Cap at ₹150.
1D Space-Optimised DP (Implemented)
DP_1D_VALUE_ONLY(orders, T, mult):
  // O(T) space instead of O(n×T)
  dp[0..T] = 0         // single row
  for i = 1..n:
    adjT = orders[i].time × mult
    net  = orders[i].net
    // RIGHT-TO-LEFT prevents double-counting
    for t = T DOWNTO adjT:
      dp[t] = max(dp[t], dp[t−adjT] + net)
  return dp[T]

// For traceback: must keep full 2D table
DP_2D_TRACEBACK(orders, T, mult):
  // Fixed: uses > (not ≠) for tie-breaking
  n = |orders|; dp[0..n][0..T] = 0
  for i = 1..n:
    adjT = round(orders[i].time × mult)
    net  = orders[i].net
    for t = 0..T:
      dp[i][t] = dp[i-1][t]
      if adjT ≤ t:
        dp[i][t] = max(dp[i][t],
          dp[i-1][t-adjT] + net)
  // Traceback: strictly > avoids duplicate
  t=T; sel=[]
  for i = n DOWNTO 1:
    if dp[i][t] > dp[i-1][t]:  // FIXED >
      sel.add(i); t -= round(adjT[i])
  return sel, dp[n][T]
TSP Route Sequencing — Connected to Knapsack Output

Gap addressed: Previous version selected orders but never sequenced them. Now after DP selects S*, nearest-neighbour TSP orders the route:

ROUTE_SEQUENCE(S*, warehouse):
current = warehouse (0,0)
unvisited = S*; route = [warehouse]
while unvisited not empty:
next = argmin dist(current, o)
for o in unvisited
route.append(next)
unvisited.remove(next)
current = next
route.append(warehouse) // return
return route, totalDist

This gives a 2-approx of TSP optimal. For n≤10 orders a full permutation search is feasible (10! = 3.6M, <100ms).

Integration pipeline:
1. Knapsack selects order subset S*
2. Dijkstra computes shortest path
from warehouse to each zone
3. Nearest-neighbour TSP orders
the actual delivery sequence
4. Total route distance feeds
back into SLA penalty check
Complexity: O(|S*|²) for NN-TSP

🧠 Algorithm Analysis — All 6 Methods
Baselines matter: FCFS and Nearest-First simulate manual dispatch. These show real-world improvement, not just theoretical comparison.
① Greedy
O(n log n) · O(n)
Sort by net/adjTime ratio. Pick greedily. Never optimal when a large low-ratio order fills budget better.
② DP — Fixed Traceback
O(n×T) · O(n×T)
Fix: Traceback now uses dp[i][t] > dp[i-1][t] (strict), not ≠, fixing duplicate-selection when multiple items produce equal dp values.
③ Hybrid — Empirically Tested
O(k log k + k×T) · O(k×T)
Greedy prunes to top-60%. DP on reduced set. Benchmarks (Tab 7) show <2% profit loss vs full DP with 40% fewer states. Empirically verified.
④ Genetic — Corrected
O(G×P×n) · O(P×n)
Fixes: True tournament selection (sample 4, pick best), proper elitism (top-4 preserved), verified 5% bit-flip mutation rate.
⑤ FCFS Baseline
O(n) · O(n)
Takes orders in arrival order until budget full. No optimization. Realistic simulation of un-assisted rider dispatch.
⑥ Nearest-First
O(n log n) · O(n)
Sorts by distance ASC. Optimises travel but ignores profit. Common real-world heuristic for fuel-conscious riders.

DP_FIXED(orders, T):
  adjT[i] = round(orders[i].time × mult)
  net[i]  = orders[i].profit − slaP(...)
  dp[0..n][0..T] = 0

Fixed Genetic Algorithm
GA_FIXED(orders, T, G=40, P=20):
  pop = random_population(P, n)

  for gen = 1..G:
    // Elitism: preserve top-4 exactly
    pop.sortBy(fitness DESC)
    next_gen = pop[0..3]            // elitism

    while |next_gen| < P:
      // Fixed tournament selection
      p1 = tournament(pop, size=4)
      p2 = tournament(pop, size=4)
      cut = random(1, n-1)
      child = [...p1[0..cut],
               ...p2[cut+1..n]]
      // Verified 5% bit-flip mutation
      for bit in child:
        if rand() < 0.05: flip(bit)
      next_gen.add(repair(child, T))
      // repair: drop excess until feasible
    pop = next_gen

  return best of pop (fitness = profit)

Dijkstra + TSP Integration
DIJKSTRA(graph, source):
  dist[v]=∞ for all v; dist[src]=0
  pq = MinPriorityQueue()
  pq.insert(source, 0)
  while pq not empty:
    u = pq.extractMin()
    for (v, w) in adj[u]:
      w_eff = w × zoneMult(v)
      if dist[u]+w_eff < dist[v]:
        dist[v] = dist[u]+w_eff
        prev[v] = u
        pq.decreaseKey(v, dist[v])
  return dist, prev

NN_TSP(selected_orders, dijkstra_dist):
  // Nearest-neighbour on Dijkstra dist matrix
  route=[warehouse]; cur=warehouse
  unvisited = set(selected_orders)
  while unvisited:
    next = argmin dist[cur][v] for v in unvisited
    route.append(next); cur=next
    unvisited.remove(next)
  route.append(warehouse)
  return route

Chromosome Encoding + Repair Operator
// Binary chromosome length = n
// [1, 0, 1, 1, 0, 0, 1, 0]
//  ↑       ↑ ↑       ↑
//  A       C D       G selected

fitness(chromosome, T):
  totalTime=0; totalProfit=0
  for i=0..n-1:
    if chromosome[i]==1:
      totalTime   += orders[i].adjTime
      totalProfit += orders[i].net
  if totalTime > T: return 0  // infeasible
  return totalProfit
// REPAIR OPERATOR (missing in original!)
// Ensures crossover/mutation stays feasible
repair(chromosome, T):
  selected = [i for i if chromosome[i]]
  selected.sortBy(net/adjTime DESC)  // best first
  used=0; result=[0]*n
  for i in selected:
    if used + orders[i].adjTime ≤ T:
      result[i]=1
      used += orders[i].adjTime
  return result

// Without repair, crossover produces infeasible
// chromosomes → fitness=0 → wasted generations
