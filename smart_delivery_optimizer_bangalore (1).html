DP recurrence (0/1 knapsack with integer-scaled time):
DP[i][t] = max(DP[i−1][t], DP[i−1][t − scaledTime[i]] + netValue[i])
where scaledTime = round(adjTime × 10) to convert decimals to integers.

Optimal substructure proof
Optimal substructure: If a set S* is the optimal solution for time budget T, then S* minus any order i is also optimal for budget T − time[i]. This follows because if a better sub-solution existed, we could substitute it into S* to improve S*, contradicting its optimality.

Overlapping subproblems: DP[i][t] is reused for all orders i+1…n, so memoization eliminates exponential recomputation. The top-down memoized variant avoids filling cells where t < scaledTime[i], reducing memory for sparse state spaces.
Dijkstra runs from Hub → each delivery zone using a weighted road graph. Route cost feeds directly into the net-value objective, making routing affect order selection itself — resolving the faculty's core criticism.

Multi-vehicle coordination (future: 2 riders)
Current scope: Single rider, time-budget knapsack.
Multi-vehicle extension: Partition order set into k subsets, one per rider, each solved with DP. Requires VRP formulation — out of scope for this prototype but architecturally ready (each rider runs an independent DP instance on their assigned subset).
Identifies the "breaking point" (faculty remark) where DP's O(n×T) memory and time exceed hardware limits. Greedy remains O(n log n) and practical at large n. Hybrid is recommended beyond the break-even threshold.

Multi-objective fitness function
Net Value = Rider Profit − SLA Penalty − λ × Dijkstra Route Cost

λ controls the profit–distance trade-off. At λ=0, reduces to pure profit maximization (standard knapsack). At λ=1, heavily penalizes long routes. Faculty suggestion addressed: dynamic weights allow simultaneous optimization of both profit maximization and distance minimization.
